# Session X0 — Walking-Skeleton Deploy

**Wave 0 · concurrent with F0 · ZERO code dependency (self-contained).**
Read `../00-master-plan.md` §7 (deployment-readiness standards — this session *is* the "X0 first (Day 0)"
bullet) and §8 (worktree/PR workflow). The whole point of X0 is to pay down the entire
image → GHCR → tunnel → TLS → secret pipeline **before any feature work, while failures are cheap** — the
audit's highest-leverage schedule insurance (`md/06` M-TIMELINE-1; `md/07` risk "[SCHEDULE,
highest-leverage]"). X0 shares **no** code with F0: it builds a self-contained skeleton under `app_skeleton/`
with its own minimal deps, so it does not depend on F0's `pyproject.toml`. **SHIP later reconciles this
skeleton to the production image** (master §3: `Dockerfile,compose  X0(skeleton) → SHIP(production)`).

## Goal

Stand up the **hosted deploy pipeline end-to-end with a trivial payload**: a multi-stage Docker image (Node
builds a placeholder SPA to static; Python serves it + `GET /health` via FastAPI), published to a public
GHCR repo, running on one always-on EC2 instance under `docker compose` + a Cloudflare Tunnel, reachable at a
real `https://…` URL. After X0 merges, "deploy" is a fixed, proven, byte-identical pull/swap for every later
stage — never a first-time integration.

## Scope (build these)

1. **Self-contained skeleton app** (`app_skeleton/`) — a trivial FastAPI app with **its own minimal deps**
   (fastapi + uvicorn only; a tiny `requirements.txt` or inline `pyproject`, *not* F0's) that:
   `GET /health` → `200` `{"status":"ok"}`; `GET /` → the placeholder SPA bundle via `StaticFiles`.
   **It MUST NOT import `chanakya/`** — X0 is independent of F0 (keeps the two Wave-0 sessions conflict-free
   and lets X0 boot before any spine code exists).
2. **Placeholder SPA** (`app_skeleton/web/`) — a minimal Node/Vite (or bare `index.html`) build emitting an
   "it boots" static page. Keep the toolchain lean; pin the Node version (build-break risk, `md/07`).
3. **Multi-stage Dockerfile** (`Dockerfile`) — Node stage builds the placeholder SPA → static assets; Python
   stage installs the skeleton's minimal deps, copies the built SPA in, runs Uvicorn serving the static
   bundle + the FastAPI route. Same shape as the production image so SHIP can extend it in place.
4. **Compose** (`docker-compose.yml`) — `restart: unless-stopped`; app port **bound to localhost** (no public
   inbound); `ANTHROPIC_API_KEY` injected from `.env` (gitignored — `.gitignore` already commits `.env`; rely
   on it, don't own it); `cloudflared` run as a compose service (or documented host daemon) dialling out to
   the tunnel.
5. **GHCR publish** — build + push the image to a **public** GHCR repo; document the exact `docker run
   ghcr.io/<you>/…` and `docker compose pull && up -d` commands in `deploy/`.
6. **EC2 + Cloudflare Tunnel** (`deploy/`) — bring-up notes for one always-on EC2 box: install Docker +
   compose, `docker compose up -d`, run `cloudflared` → a real `https://…` URL with managed TLS, **no domain,
   no DNS, no open inbound security-group ports** (the tunnel dials out; only inbound is SSH/SSM). Record the
   fallback ladder pointer (Caddy+domain → bare-IP → SSH tunnel; `md/07` "Hosting architecture") — one line,
   don't build it.
7. **Prove it live** — confirm the public https tunnel URL loads the "it boots" page and `GET /health`
   returns `200` from both the GHCR image and the EC2 deploy.

## Contracts implemented
Master **§7** (deployment-readiness standards) — the "X0 first (Day 0)" bullet in full. X0 does **not** freeze
any code contract; it owns the *skeleton* `Dockerfile`/`docker-compose.yml`/`deploy/` that SHIP extends to
production (master §3 ownership: `X0(skeleton) → SHIP(production)`). No F0-frozen surface is touched.

## Acceptance criteria
- [ ] A hosted **"it boots" page** is reachable over the **https Cloudflare Tunnel URL** (real TLS, no domain,
      no open inbound ports).
- [ ] The **public GHCR image pulls and runs** (`docker run ghcr.io/<you>/…` serves the page locally,
      byte-identical to the EC2 box).
- [ ] `GET /health` returns **`200`** locally, from the GHCR image, and over the tunnel URL.
- [ ] The whole **image → GHCR → `docker compose` on EC2 → tunnel** pipeline is proven working while failures
      are cheap (before any feature work).
- [ ] `app_skeleton/` imports **no** `chanakya/` code and carries its own minimal deps; the image builds with
      no dependency on F0's `pyproject.toml`.
- [ ] `.env` stays gitignored and out of the image/logs; `ANTHROPIC_API_KEY` is injected via env, never baked.

## Owned paths (nothing else)
`Dockerfile`, `docker-compose.yml`, `deploy/**` (cloudflared config + EC2 bring-up notes), `app_skeleton/**`
(the self-contained FastAPI skeleton + placeholder SPA). **Depends on:** none. **LLM:** no.

## Out of scope
The real app and the real React/Vite SPA; wiring FastAPI to `chanakya/`; baking `config/` + corpus +
seeded SQLite baseline into the image; `make` targets; rollback drill; the Bedrock instance-role path — **all
that is SHIP** (master §3), which reconciles this skeleton into the production image. `.github/workflows`,
`pyproject.toml`, `Makefile` are F0-owned; X0 does not touch them.
